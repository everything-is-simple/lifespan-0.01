# 系统级路线图与进度跟踪器规格

日期：`2026-04-09`
状态：`生效中`

## 目标

冻结系统级路线图文档应该包含哪些栏目、模块状态如何定义、何时需要刷新。

## 权威文档

本规格对应的当前权威路线图为：

- `docs/02-spec/Ω-system-delivery-roadmap-20260409.md`

## 路线图必须包含的栏目

系统级路线图至少必须包含以下内容：

1. 当前系统进度
2. 系统阶段划分
3. 各模块状态
4. 下一锤
5. 阻塞项
6. 里程碑定义

## 模块状态枚举

模块状态统一使用以下 6 个离散值：

1. `未开始`
2. `设计中`
3. `建设中`
4. `局部可验`
5. `主线待接`
6. `主线已接`

## 状态含义

### `未开始`

- 只有方向，没有正式 design/spec/card

### `设计中`

- 已有 design/spec 或正式执行卡
- 但还没有稳定表族、runner 或正式出口

### `建设中`

- 已开始正式建设
- 但还没有形成可复验的 bounded 结果

### `局部可验`

- 已有局部表族、runner、bounded evidence
- 但还没达到能接系统主线的程度

### `主线待接`

- 模块本身已基本成型
- 但与上下游正式桥接仍未完全接好

### `主线已接`

- 已进入当前冻结主链
- 且具备可复验、可解释、可继续增量推进的正式入口

## 当前进度栏要求

“当前进度”必须至少回答：

1. 当前整体处在哪个系统阶段
2. 已完成哪些地基建设
3. 下一阶段准备进入什么模块

## 下一锤栏要求

“下一锤”必须明确：

1. 当前优先模块
2. 当前优先产物
3. 不先做它的风险

## 阻塞项栏要求

阻塞项必须是会直接影响推进顺序的现实问题，而不是泛泛风险。

阻塞项建议按以下三类组织：

1. 契约未冻结
2. 正式账本未建立
3. 上下游桥接未完成

## 里程碑要求

里程碑必须同时写清楚：

1. 名称
2. 判定条件
3. 当前状态

## 刷新时机

出现以下情况时，路线图应同步刷新：

1. 当前待施工卡切换到新阶段
2. 某个模块状态跨级变化
3. 下一锤发生变化
4. 系统主线冻结发生变化

## 与入口文件联动

路线图或其设计/规格发生正式变化时，应视为仓库正式口径变化，需同步刷新：

1. `AGENTS.md`
2. `README.md`
3. `pyproject.toml`
