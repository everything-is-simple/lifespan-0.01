# 系统级路线图与进度跟踪器规格

日期：`2026-04-09`
状态：`生效中`

## 目标

冻结系统级路线图文档应该包含哪些栏目、模块状态如何定义、何时需要刷新。

## 权威文档

本规格对应的当前权威路线图为：

- `docs/02-spec/Ω-system-delivery-roadmap-20260409.md`

## 路线图必须包含的栏目

系统级路线图至少必须包含以下内容：

1. 当前系统进度
2. 系统阶段划分
3. 各模块状态
4. 下一锤
5. 阻塞项
6. 里程碑定义
7. 老仓来源分层
8. 模块来源与继承方式
9. 当前不敢写死的点

## 模块状态枚举

模块状态统一使用以下 6 个离散值：

1. `未开始`
2. `设计中`
3. `建设中`
4. `局部可验`
5. `主线待接`
6. `主线已接`

## 模块继承方式枚举

路线图对每个模块至少还应补一列“继承方式”，统一使用以下 4 个离散值：

1. `沿袭为主`
2. `沿袭后改写`
3. `只吸收经验`
4. `全新设计`

含义：

- `沿袭为主`
  - 老仓已有成体系 design/spec/execution 结论，新系统主要沿袭其裁决
- `沿袭后改写`
  - 老仓结论可保留，但正式账本、命名或边界需要重写
- `只吸收经验`
  - 老仓更多提供踩坑与桥接教训，不能直接当新系统正式合同
- `全新设计`
  - 老仓只有想法或边界，缺少可直接沿袭的正式实现遗产

## 模块置信度枚举

路线图对每个模块至少还应补一列“置信度”，统一使用以下 3 个离散值：

1. `高`
2. `中`
3. `低`

含义：

- `高`
  - 同时具备历史 design/spec/execution 多层材料，且已有明确结论可迁移
- `中`
  - 有部分正式材料与结论，但新系统正式合同仍需大量改写
- `低`
  - 主要只有边界想法、经验提示或零散材料，尚不足以支撑直接施工

## 状态含义

### `未开始`

- 只有方向，没有正式 design/spec/card

### `设计中`

- 已有 design/spec 或正式执行卡
- 但还没有稳定表族、runner 或正式出口

### `建设中`

- 已开始正式建设
- 但还没有形成可复验的 bounded 结果

### `局部可验`

- 已有局部表族、runner、bounded evidence
- 但还没达到能接系统主线的程度

### `主线待接`

- 模块本身已基本成型
- 但与上下游正式桥接仍未完全接好

### `主线已接`

- 已进入当前冻结主链
- 且具备可复验、可解释、可继续增量推进的正式入口

## 当前进度栏要求

“当前进度”必须至少回答：

1. 当前整体处在哪个系统阶段
2. 已完成哪些地基建设
3. 下一阶段准备进入什么模块

## 下一锤栏要求

“下一锤”必须明确：

1. 当前优先模块
2. 当前优先产物
3. 不先做它的风险
4. 它主要继承自哪些老仓来源

## 阻塞项栏要求

阻塞项必须是会直接影响推进顺序的现实问题，而不是泛泛风险。

阻塞项建议按以下三类组织：

1. 契约未冻结
2. 正式账本未建立
3. 上下游桥接未完成

## 里程碑要求

里程碑必须同时写清楚：

1. 名称
2. 判定条件
3. 当前状态
4. 当前主要依赖哪些上游模块先完成

## 老仓来源分层栏要求

“老仓来源分层”必须至少回答：

1. 哪些模块属于核心已验证模块
2. 哪些模块只有支持性经验
3. 哪些模块研究偏少
4. 哪些模块属于新系统正式新建边界

## 模块来源与继承方式栏要求

路线图中的模块表至少必须补齐以下字段：

1. 模块名
2. 当前状态
3. 主要来源
4. 继承方式
5. 置信度
6. 下一步重点

“主要来源”至少应写到目录级别，而不是只写“参考老系统”。

## 当前不敢写死的点栏要求

这部分必须只写真正影响施工判断的未定项，而不是泛泛风险。

至少应覆盖：

1. 当前尚未冻结的正式表族
2. 当前尚未收口的桥接合同
3. 当前仍只有边界、没有正式表合同的模块

## 刷新时机

出现以下情况时，路线图应同步刷新：

1. 当前待施工卡切换到新阶段
2. 某个模块状态跨级变化
3. 下一锤发生变化
4. 系统主线冻结发生变化
5. 某个模块的主要来源判断或继承方式发生变化
6. 某个低置信度模块被正式抬升为可施工模块

## 与入口文件联动

路线图或其设计/规格发生正式变化时，应视为仓库正式口径变化，需同步刷新：

1. `AGENTS.md`
2. `README.md`
3. `pyproject.toml`
