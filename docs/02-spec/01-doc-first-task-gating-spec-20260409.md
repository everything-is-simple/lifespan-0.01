# 文档先行任务门禁规格

日期：`2026-04-09`
状态：`生效`

## 适用范围

本规格适用于：

- 新模块
- 新数据库
- Schema 变更
- Pipeline 新增
- 正式代码生成
- 有行为影响的代码改写

## 代码前置输入

在代码生成开始前，任务必须具备：

1. 需求
2. 设计
3. 任务分解

## 允许形式

需求、设计和任务分解可以分散在多个正式文档中，但它们必须能从执行卡上被明确找到。

## 执行门禁

只要执行卡没有链接到以下内容，就不应进入正式代码实现：

- 至少一份设计文档
- 至少一份规格文档或明确契约说明
- 一份实现切片清单

## 后续执行方向

本仓库未来应通过检查器自动化这条规则，但在自动化出现之前，治理规则已经立即生效。
