# 执行入口

`docs/03-execution/` 是正式执行闭环区域。

它不是随手记笔记的地方，而是正式实现任务的收口区。

## 执行闭环

正式闭环是：

`需求 -> 设计 -> 任务分解 -> 卡片 -> 实现 -> 证据 -> 记录 -> 结论`

## 默认阅读顺序

第一次进入本目录时，默认按下面顺序阅读：

1. `00-conclusion-catalog-20260409.md`
2. `00-card-execution-discipline-20260409.md`
3. `00-execution-reading-order-20260409.md`

## 默认动作

如果你只是想知道“现在正式成立了什么”，先看：

1. `00-conclusion-catalog-20260409.md`
2. 对应结论文件

如果你想继续正式实现，先看：

1. 当前活动执行卡
2. 卡片链接的需求、设计、任务分解
3. 再开始实现

## 硬规则

如果执行卡没有明确指向以下内容，就不允许进行正式代码变更：

1. 需求
2. 设计
3. 任务分解

## 目录职责

- 根目录
  - 放卡片、结论、索引、模板
- `evidence/`
  - 放测试证据、命令证据、运行证据
- `records/`
  - 放执行记录、实现轨迹、收口说明

## 收口纪律

任何正式实现任务，缺少以下任意一件，都不能算完成：

1. `card`
2. `evidence`
3. `record`
4. `conclusion`
