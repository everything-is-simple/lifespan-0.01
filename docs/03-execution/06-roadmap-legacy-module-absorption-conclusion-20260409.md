# 路线图老仓来源吸收与系统总控增强 结论

结论编号：`06`
日期：`2026-04-09`
状态：`生效中`

## 裁决

- 接受：系统级路线图从本轮开始不再只是阶段看板，而是正式承担“老仓来源 grounding、模块继承方式、模块置信度与当前未定项”的裁决职责。
- 接受：新仓已经有一份单独的参考图谱，可用于持续追溯 `position / alpha / malf / data / system / trade / core / structure / filter / portfolio_plan` 的老仓来源。
- 拒绝：把所有模块都当成“同等成熟、同等可直接迁入”的想法。

## 原因

1. `position / alpha / malf` 的老仓材料明显最扎实，已经可以作为新系统下一阶段正式设计的主要吸收对象。
2. `data / system` 有大量桥接、验收与修复经验，但仍需新系统重写正式合同表达。
3. `trade / core` 的正式实验和账本沉淀相对较薄，只能优先吸收经验，不宜假装已经 ready。
4. `structure / filter / portfolio_plan` 当前主要属于新系统正式新建边界，必须在吸收旧想法后重新建立正式合同。

## 影响

1. 以后你每天读 `Ω-system-delivery-roadmap-20260409.md` 时，看到的不只是“现在做到哪”，还会同时看到“这个判断是建立在哪些老仓来源上”。
2. 后续开启 `position` 正式 design/spec 时，已经有一份系统级依据说明为什么它应该先于其他业务模块开工。
3. 新仓对老系统的吸收方式从“聊天中口头记忆”升级为“仓库内正式可追溯图谱”。
