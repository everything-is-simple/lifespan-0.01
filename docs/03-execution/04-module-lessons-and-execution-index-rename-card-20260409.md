# 模块经验沉淀与执行入口字母化

卡片编号：`04`
日期：`2026-04-09`
状态：`已完成`

## 需求

- 问题：
  老系统已经用真实代价换出一批跨模块边界教训，但新仓还没有把这些经验沉淀进各模块文档；同时 `03-execution` 下最关键的三份入口文件仍使用普通数字序号，辨识度不够高。
- 目标结果：
  提炼老系统总表中的模块经验，写入新仓各模块设计文档；并把执行入口中的阅读顺序、卡目录、完工账本改成更醒目的字母命名，同时修复全仓引用。
- 为什么现在做：
  下一步就要正式开 `position` 的模块卡，如果不先把边界经验和执行入口整理好，后续继续接主线时仍然容易重复踩坑，也容易在执行区入口里迷失。

## 设计输入

- 设计文档：`docs/01-design/00-system-charter-20260409.md`
- 规格文档：`docs/02-spec/00-repo-layout-and-docflow-spec-20260409.md`

## 任务分解

1. 从老系统“全模块踩坑总表”提炼可复用边界，落入新仓 `docs/01-design/modules/`。
2. 为新仓补模块设计入口，并把入口阅读顺序同步到仓库主入口。
3. 把 `03-execution` 的 3 个核心入口文件改成字母命名，并修复脚本、测试和文档引用。
4. 跑治理与测试，回填证据、记录和结论。

## 实现边界

- 范围内：
  - 模块经验文档
  - 执行入口文件重命名
  - 治理脚本与引用修复
  - 最小验证
- 范围外：
  - `position` 资金管理正式设计
  - 业务代码迁移
  - 历史老仓文档批量搬运

## 收口标准

1. 模块经验文档落地
2. A/B/C 执行入口文件命名完成
3. 脚本与测试引用修复并通过
4. 证据、记录、结论写完
